Knots

Source history beyond artifacts.

Knots is a grammar-agnostic structured source versioning engine for human+agent co-authoring. It tracks what was written, how it evolved, and where it lives in a document—without treating the mutable file as canonical state.

View repository Read the knots memo

Intended as the landing page content for knots.cyphernetics.io.

Substrate

Stable semantic units

Knots turns paragraphs, headings, list items, citations, and other grammar-aware blocks into addressable objects with durable identity. The source becomes pointable.

History

Derivation and composition as separate axes

Rewriting and rearranging are different operations. Knots records both: how content changes, and how content is placed, moved, or removed inside a document.

Publishing

Rendered outputs with provenance

Markdown, knots-markdown, and other projections are derived artifacts. The sealed history, publish refs, and build provenance remain inspectable upstream of the surface.

Why it matters

Model collaboration gets stronger when source is structured enough to navigate deterministically.

Current agent interfaces are good at generating text and bad at preserving source state. Chat becomes the dominant artifact; the author points indirectly; context drifts.

Knots provides the missing substrate: stable block identity, historical addressability, render surfaces, and a source state that can be queried, revised, and published without collapsing into transcript theater.

ktui

The interface should remain calm enough to read, but precise enough to point the model at exact source units.

ktui is the interface frontier built on top of this substrate: a fast, sparse authoring harness where structural traversal, reading, and model composition reinforce each other instead of competing for space. Knots is what makes that loop real.
